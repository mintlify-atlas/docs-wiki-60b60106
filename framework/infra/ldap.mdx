`Volo.Abp.Ldap.Abstractions` exposes the contract; `Volo.Abp.Ldap` provides an implementation backed by [LdapForNet](https://github.com/flamencist/ldap4net). Settings drive the connection, so they can be overridden per tenant.

## ILdapManager

```csharp framework/src/Volo.Abp.Ldap.Abstractions/Volo/Abp/Ldap/ILdapManager.cs
public interface ILdapManager
{
    Task<bool> AuthenticateAsync(string username, string password);
}
```

The implementation creates a connection, binds with the supplied credentials, and reports success/failure. Exceptions are logged but never propagate:

```csharp framework/src/Volo.Abp.Ldap/Volo/Abp/Ldap/LdapManager.cs
public virtual async Task<bool> AuthenticateAsync(string username, string password)
{
    try
    {
        using (var conn = await CreateLdapConnectionAsync())
        {
            await AuthenticateLdapConnectionAsync(conn, username, password);
            return true;
        }
    }
    catch (Exception ex)
    {
        Logger.LogException(ex);
        return false;
    }
}
```

`ConfigureLdapConnectionAsync`, `ConnectAsync`, and `AuthenticateLdapConnectionAsync` are all `virtual` so you can plug in TLS, custom auth types, or different bind DN strategies. The default uses `Native.LdapAuthType.Simple`.

## Settings

`LdapSettingNames` defines every key fed into the `LdapManager` through `ILdapSettingProvider`:

```csharp framework/src/Volo.Abp.Ldap.Abstractions/Volo/Abp/Ldap/LdapSettingNames.cs
public const string Ldaps = "Abp.Ldap.Ldaps";
public const string ServerHost = "Abp.Ldap.ServerHost";
public const string ServerPort = "Abp.Ldap.ServerPort";
public const string BaseDc    = "Abp.Ldap.BaseDc";
public const string Domain    = "Abp.Ldap.Domain";
public const string UserName  = "Abp.Ldap.UserName";
public const string Password  = "Abp.Ldap.Password";
```

`LdapSettingDefinitionProvider` (in `Volo.Abp.Ldap`) materializes these as ABP setting definitions; `LdapSettingProvider` reads them through `ISettingProvider` so tenant-level overrides apply automatically.
