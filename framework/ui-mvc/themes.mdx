An ABP *theme* is an MVC module that owns the layout `.cshtml` files, the chrome view components (top navbar, sidebar, footer, toolbar slots), and the script/style bundles that turn the framework's tag helpers into a coherent visual identity. The framework defines the contract — `ITheme`, `IThemeManager`, `AbpThemingOptions`, standard layout names — and ships `Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared` with the cross-theme assets. Concrete themes (Basic, LeptonX, custom) plug into that contract.

## The contract

From `framework/src/Volo.Abp.AspNetCore.Mvc.UI/Volo/Abp/AspNetCore/Mvc/UI/Theming/`:

```csharp
public interface ITheme
{
    string GetLayout(string name, bool fallbackToDefault = true);
}

public interface IThemeManager { ITheme CurrentTheme { get; } }
public interface IThemeSelector { ThemeInfo GetCurrentThemeInfo(); }

public class AbpThemingOptions
{
    public ThemeDictionary Themes { get; }
    public string? DefaultThemeName { get; set; }
    public string? BaseUrl { get; set; }
}
```

`StandardLayouts.cs` enumerates the four canonical layout names that every theme should implement: `Application`, `Account`, `Public`, `Empty`. `ThemeNameAttribute.cs` is how a theme advertises its name — `ThemeInfo` reads it via reflection during registration.

## Theme.Shared: assets every theme inherits

`framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/AbpAspNetCoreMvcUiThemeSharedModule.cs` lays the foundation:

```csharp
[DependsOn(
    typeof(AbpAspNetCoreMvcUiBootstrapModule),
    typeof(AbpAspNetCoreMvcUiPackagesModule),
    typeof(AbpAspNetCoreMvcUiWidgetsModule),
    typeof(AbpFeaturesModule)
)]
public class AbpAspNetCoreMvcUiThemeSharedModule : AbpModule
{
    public override void ConfigureServices(ServiceConfigurationContext context)
    {
        Configure<AbpVirtualFileSystemOptions>(options =>
        {
            options.FileSets.AddEmbedded<AbpAspNetCoreMvcUiThemeSharedModule>("Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared");
        });

        Configure<AbpBundlingOptions>(options =>
        {
            options.StyleBundles
                .Add(StandardBundles.Styles.Global, bundle => bundle.AddContributors(typeof(SharedThemeGlobalStyleContributor)));
            options.ScriptBundles
                .Add(StandardBundles.Scripts.Global, bundle => bundle.AddContributors(typeof(SharedThemeGlobalScriptContributor)));
        });
    }
}
```

What it contributes:

- `StandardBundles.Styles.Global` and `StandardBundles.Scripts.Global` — the bundle names that every theme extends. Modules drop contributors here to add CSS/JS that should appear in every page.
- Embedded `Views/Error/` razor views (`AbpErrorViewModel`, `AbpErrorPageOptions`) that `app.UseErrorPage()` mounts.
- `Toolbars/` — `IToolbarContributor`, `AbpToolbarOptions`, `StandardToolbars.Main` / `User`. Page-level toolbars live in `PageToolbars/`.
- `Controllers/` — shared utility controllers (e.g. `AbpRouteController`, language-switch controller, antiforgery token endpoint).
- Embedded packages from `Volo.Abp.AspNetCore.Mvc.UI.Packages` — Bootstrap CSS, Tempus Dominus, malihu scrollbar, etc., served via the virtual file system.

Every concrete theme `[DependsOn(typeof(AbpAspNetCoreMvcUiThemeSharedModule))]` so it inherits all of the above.

## A concrete theme: BasicTheme

`modules/basic-theme/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Basic/BasicTheme.cs`:

```csharp
[ThemeName(Name)]
public class BasicTheme : ITheme, ITransientDependency
{
    public const string Name = "Basic";

    public virtual string GetLayout(string name, bool fallbackToDefault = true)
    {
        switch (name)
        {
            case StandardLayouts.Application: return "~/Themes/Basic/Layouts/Application.cshtml";
            case StandardLayouts.Account:     return "~/Themes/Basic/Layouts/Account.cshtml";
            case StandardLayouts.Empty:       return "~/Themes/Basic/Layouts/Empty.cshtml";
            default: return fallbackToDefault ? "~/Themes/Basic/Layouts/Application.cshtml" : null;
        }
    }
}
```

The `~/Themes/Basic/Layouts/*.cshtml` files are embedded in the assembly and resolved from the virtual file system, not the host project. That is what makes themes drop-in: a host changes a single `[DependsOn(...)]` from `BasicTheme` to a different theme module and every layout switches without any view scaffolding changes.

## Theme module registration

`AbpAspNetCoreMvcUiBasicThemeModule` is representative of how a theme registers itself:

```csharp
[DependsOn(typeof(AbpAspNetCoreMvcUiThemeSharedModule),
           typeof(AbpAspNetCoreMvcUiMultiTenancyModule))]
public class AbpAspNetCoreMvcUiBasicThemeModule : AbpModule
{
    public override void ConfigureServices(ServiceConfigurationContext context)
    {
        Configure<AbpThemingOptions>(options =>
        {
            options.Themes.Add<BasicTheme>();
            if (options.DefaultThemeName == null)
            {
                options.DefaultThemeName = BasicTheme.Name;
            }
        });

        Configure<AbpVirtualFileSystemOptions>(options =>
        {
            options.FileSets.AddEmbedded<AbpAspNetCoreMvcUiBasicThemeModule>(
                "Volo.Abp.AspNetCore.Mvc.UI.Theme.Basic");
        });

        Configure<AbpToolbarOptions>(options =>
        {
            options.Contributors.Add(new BasicThemeMainTopToolbarContributor());
        });

        Configure<AbpBundlingOptions>(options =>
        {
            options.StyleBundles
                .Add(BasicThemeBundles.Styles.Global, bundle =>
                    bundle.AddBaseBundles(StandardBundles.Styles.Global)
                          .AddContributors(typeof(BasicThemeGlobalStyleContributor)));
        });
    }
}
```

Four steps that any theme repeats:

1. **Register the `ITheme` class** via `AbpThemingOptions.Themes.Add<T>()` and optionally set `DefaultThemeName` (only the first theme module to set it wins).
2. **Mount embedded views** through the virtual file system so `~/Themes/Basic/Layouts/Application.cshtml` resolves.
3. **Contribute toolbars** via `IToolbarContributor` — typically a left-side menu and a right-side user-menu/language-switcher block — registered with `AbpToolbarOptions`.
4. **Define theme-scoped bundles** that `AddBaseBundles(StandardBundles.Styles.Global)` so the shared assets always load first.

## Toolbars

A theme layout calls `<abp-toolbar name="@StandardToolbars.Main" />` (a tag helper from Theme.Shared) inside the navbar. That tag helper resolves `IToolbarManager`, which iterates the contributors registered in `AbpToolbarOptions` for the matching toolbar name. Each contributor mutates the `Toolbar.Items` collection (adding view components by type). The basic theme's `BasicThemeMainTopToolbarContributor` for example adds the `LanguageSwitch` and `UserMenu` view components.

This is the same shape as `IMenuContributor` for navigation (see [Navigation](/framework/ui-mvc/navigation)) — it lets feature modules attach to the shell without owning it.

## Layout discovery in views

The standard `_ViewStart.cshtml` of an ABP MVC project reads the layout dynamically:

```cshtml
@using Volo.Abp.AspNetCore.Mvc.UI.Theming
@inject IThemeManager ThemeManager
@{ Layout = ThemeManager.CurrentTheme.GetLayout(StandardLayouts.Application); }
```

When a page wants the chromeless Account layout (login, register), it sets `Layout = ThemeManager.CurrentTheme.GetLayout(StandardLayouts.Account)` in its own `@{ ... }` block. The same razor file works against any theme module the host plugs in.

## Server-side Blazor parallel

The same `IThemeManager`/`ITheme` contract powers the Blazor Server theme system (`Volo.Abp.AspNetCore.Components.Server.Theming`) — see [Blazor Server](/framework/blazor/server). The basic theme has parallel `Volo.Abp.AspNetCore.Components.Server.BasicTheme` / `Volo.Abp.AspNetCore.Components.WebAssembly.BasicTheme` packages that register a Blazor-shaped `ITheme` against the same options. That is how a single feature module ships Razor pages, Blazor Server components, and Blazor WebAssembly components against three different chromes without forking the contract.

For asset registration mechanics see [Bundling](/framework/aspnetcore/bundling); for the chrome-internal navigation primitives see [Navigation](/framework/ui-mvc/navigation).
