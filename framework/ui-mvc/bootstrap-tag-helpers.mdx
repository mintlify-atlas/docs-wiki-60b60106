`Volo.Abp.AspNetCore.Mvc.UI.Bootstrap` wraps Bootstrap 5 markup behind Razor tag helpers so layouts stay declarative. Every helper is a normal `Microsoft.AspNetCore.Razor.TagHelpers.TagHelper` paired with an injectable `IAbpTagHelperService<T>` that does the rendering — the split is what makes them testable and DI-friendly. The base classes (`AbpTagHelper`, `AbpTagHelperService`, `IAbpTagHelperLocalizer`) live at the root of `framework/src/Volo.Abp.AspNetCore.Mvc.UI.Bootstrap/TagHelpers/`.

This page enumerates the tag helper categories shipped by the package. Each row is grounded in a real `*TagHelper.cs` source file under `framework/src/Volo.Abp.AspNetCore.Mvc.UI.Bootstrap/TagHelpers/<Category>/`.

## Forms

`TagHelpers/Form/`

| Tag helper | Element | Description |
| --- | --- | --- |
| `AbpInputTagHelper` | `<abp-input>` | Renders a labelled `<input>` with model-binding, validation, info text, format control, and disabled/readonly handling. Drives most form inputs in ABP UIs. |
| `AbpSelectTagHelper` | `<abp-select>` | Renders a labelled `<select>` from `SelectItems` or an enum-bound model property. |
| `AbpRadioInputTagHelper` | `<abp-radio>` | Renders a labelled radio group from an enum or `AbpRadioButton` list. |
| `AbpDynamicformTagHelper` | `<abp-dynamic-form>` | Reflects over a model and emits a complete form (label + input + validation) for each property using `AbpInputTagHelper` and friends. Honours `DynamicFormIgnore`, `DisplayOrder`. |
| `AbpFormContentTagHelper` | `<abp-form-content />` | Placeholder injected by `AbpDynamicformTagHelper` to mark where custom child content should land. |
| `AbpIdNameTagHelper` | `[abp-id-name]` attribute | Auto-generates stable `id`/`name` on any HTML element from a model expression. |
| `AbpValidationAttributeTagHelper` | `[asp-validation-for]` / `[asp-validation-summary]` | Adds Bootstrap classes and `aria-*` to MVC's stock validation helpers. |

The `TagHelpers/Form/DatePicker/` subfolder ships the `<abp-input type="date">` and `<abp-input type="datetime-local">` rendering for the bundled Tempus Dominus integration. Helpers like `Placeholder.cs`, `ReadOnlyInput.cs`, `DisabledInput.cs`, and `FormGroupItem.cs` are value objects used by the form helpers above.

## Toolbar (button toolbar + table toolbar)

The page-level "toolbar" abstraction (top toolbar, page toolbar) lives in `Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/Toolbars/` and is rendered via view components, not tag helpers. Inside a Razor page the *button* toolbar is `<abp-button-toolbar>` (see Button row above).

## Implementation notes

- Every helper has a sibling `*TagHelperService.cs` that contains the rendering logic. The DI split is what lets a host override behaviour: register a derived service via `Configure<MvcOptions>` and the tag helper picks it up automatically.
- The shared `IAbpTagHelperLocalizer` (`AbpTagHelperLocalizer.cs`) is what resolves the display name from `ModelExpression` metadata — set localization on your DTO with `[DisplayName(...)]` or via a Localization Resource and every form helper picks it up.
- Enums like `FontIconType`, `HtmlHeadingType`, `AbpCardImagePosition`, `AbpModalSize`, `AbpFormControlSize`, and `AbpCardBackgroundType` are the central styling vocabulary — `*Extensions.cs` companions convert them to Bootstrap class names.

For how these helpers are composed into themes, see [Themes](/framework/ui-mvc/themes); for the unrelated *script/style* tag helpers that load assets, see [Bundling](/framework/aspnetcore/bundling).
