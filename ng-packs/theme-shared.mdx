`@abp/ng.theme.shared` (`npm/ng-packs/packages/theme-shared/`) is the layer between `@abp/ng.core` and the visual themes (`theme-basic`, LeptonX, custom themes). It owns toasts, confirmation dialogs, modal infrastructure, error pages, and the UX-side handlers that listen to `HttpErrorReporterService`.

## ToasterService

`services/toaster.service.ts`:

```typescript
@Injectable({ providedIn: 'root' })
export class ToasterService implements ToasterContract {
  private toasts$ = new ReplaySubject<Toaster.Toast[]>(1);
  private containerComponentRef!: ComponentRef<ToastContainerComponent>;

  info(message: LocalizationParam, title?: LocalizationParam, options?: Partial<Toaster.ToastOptions>)
       : Toaster.ToasterId { /* тАж */ }
  // success / warn / error / remove / clearAll
}
```

It uses `ContentProjectionService` from `@abp/ng.core` (`PROJECTION_STRATEGY.AppendComponentToBody`) to mount a single `ToastContainerComponent` on the document body and feed it through `toasts$`. Messages accept `LocalizationParam` so `'AbpUi::Saved'` keys are resolved automatically.

## ConfirmationService

`services/confirmation.service.ts` follows the same projection pattern but opens a single `ConfirmationComponent`:

```typescript
@Injectable({ providedIn: 'root' })
export class ConfirmationService {
  info(message, title, options?)    : Observable<Confirmation.Status> { /* тАж */ }
  success(message, title, options?) : Observable<Confirmation.Status>
  warn(message, title, options?)    : Observable<Confirmation.Status>
  error(message, title, options?)   : Observable<Confirmation.Status>
}
```

It returns an `Observable<Confirmation.Status>` you subscribe to instead of using a callback API.

## Modal infrastructure

`components/modal/`:

```
modal.component.ts        // <abp-modal [(visible)] size, busy, options, тАж>
modal.component.html
modal-ref.service.ts      // injected by modal content to control its host
modal-close.directive.ts  // [abp-close] hooks any button up to dismiss
```

Module dialogs (create/edit forms, permission management, feature management) declare `<abp-modal>` and let `ModalRef` drive open/close. `[abp-close]` on a button (or `modalCloseDirective`) emits the close intent without the child component needing a reference to its parent тАФ handy for re-used form components.

## Error component & handlers

`HttpErrorWrapperComponent` (`components/http-error-wrapper/`) renders the full-screen 401/403/404/500 page. The `handlers/` and per-status services hook into `HttpErrorReporterService.reporter$` and decide what to do:

- `authentication-error-handler.service` тЖТ triggers re-login.
- `tenant-resolve-error-handler.service` тЖТ handles `__tenant` resolution failure.
- `status-code-error-handler.service` / `unknown-status-code-error-handler.service` тЖТ toast or open the error wrapper page.

`abp-format-error-handler.service` overrides Angular's default `ErrorHandler` to also report through the same pipe so uncaught exceptions surface in toasts during dev.

## Extensible-table integration

`@abp/ng.theme.shared` exposes the **extensions** API (`tokens/`, `models/extensions`) that backs the extensible-table/form components in `@abp/ng.components/extensible`. Modules call `extensions.addPropContributors(...)` / `addActionContributors(...)` at module-config time, and the rendered table/form picks the metadata up automatically.

## Page alerts, menus, navbar primitives

- `PageAlertService` тАФ banner-style alerts (info/warning/error) at the top of a page.
- `NavItemsService` / `UserMenuService` тАФ registries for the side menu and the user dropdown. Themes (`theme-basic`, LeptonX) iterate these to render the chrome.
- `AbstractMenuService` тАФ base class custom menus extend.
