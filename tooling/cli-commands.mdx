This page enumerates every command shipped with the ABP CLI. Each entry maps a verb you type after `abp` (e.g. `abp new ...`) to the `IConsoleCommand` class that implements it. The bindings come from `AbpCliCoreModule.ConfigureServices` in `framework/src/Volo.Abp.Cli.Core/Volo/Abp/Cli/AbpCliCoreModule.cs`; the descriptions and options are extracted from each command's `GetUsageInfo()` method.

All command files live under `framework/src/Volo.Abp.Cli.Core/Volo/Abp/Cli/Commands/`.

## Tooling, MCP, REPL

| Command | Class | Description | Key options |
| --- | --- | --- | --- |
| `suite` | `SuiteCommand.cs` | Manage ABP Suite as a dotnet global tool. | sub-args `install`, `update`, `remove`; `--preview`, `--version` |
| `cli` | `CliCommand.cs` | Self-management for the `abp` CLI itself. | sub-args `update`, `remove`; `--preview`, `--version` |
| `mcp` | `McpCommand.cs` | Run the ABP Model Context Protocol server (stdio JSON-RPC). | sub-arg `get-config` (prints client JSON config) |
| `prompt` | `PromptCommand.cs` | Enter an interactive REPL that re-parses each typed line through `CliService`. | _(no options)_ |
| `help` | `HelpCommand.cs` | List all commands or print one command's `GetUsageInfo()`. | `<command-name>` positional |

`HelpCommand.GetUsageInfo()` is what you see when running `abp` with no arguments. It iterates `AbpCliOptions.Commands`, calls the static `GetShortDescription()` on each command type via reflection, and prints them sorted alphabetically.

## Adding a new command

Because the dispatcher is a plain dictionary, any module loaded into the CLI process can append entries:

```csharp
[DependsOn(typeof(AbpCliCoreModule))]
public class MyCliExtensionsModule : AbpModule
{
    public override void ConfigureServices(ServiceConfigurationContext context)
    {
        Configure<AbpCliOptions>(options =>
        {
            options.Commands["my-cmd"] = typeof(MyCommand);
        });
    }
}
```

See [CLI overview](/tooling/cli-overview) for the dispatch flow and [Service proxying](/tooling/service-proxying) for a deeper dive into `generate-proxy`/`remove-proxy`.
