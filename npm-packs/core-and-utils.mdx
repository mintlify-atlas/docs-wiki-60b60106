These are the two foundation packages under `npm/packs/`. Every MVC and Blazor
Server theme depends ŌĆö directly or transitively ŌĆö on `@abp/core`, and
`@abp/utils` is a one-class TypeScript helper used by the Angular packages
under `npm/ng-packs/`.

`wwwroot/libs/abp/core/`. Every other UI bundle pulls them in via the layout's
`@abpScripts` / `@abpStyles` bundles.

### What abp.js declares

`npm/packs/core/src/abp.js` builds a single `abp` global from a series of
sections (the file uses `/* SECTION ********** */` banners). Reading the source
top to bottom:

| Section | Globals it adds | Notes |
| --- | --- | --- |
| Application paths | `abp.appPath`, `abp.pageLoadTime`, `abp.toAbsAppPath(path)` | Reads `<base href>` to seed `appPath`. |
| Logging | `abp.log.{debug,info,warn,error,fatal}`, `abp.log.levels` | Thin `console.log` wrapper, level-filtered. |
| Localization | `abp.localization.{values,resources,localize,getResource,getLanguagesMap}`, `abp.localization.currentCulture` | Multi-resource lookup with `defaultResourceName` fallback. |
| Authorization | `abp.auth.{grantedPolicies,isGranted,isAnyGranted,areAllGranted,setToken,getToken,clearToken}` | Token cookie name is `Abp.AuthToken`. |
| Settings | `abp.setting.{values,get,getBoolean,getInt}` | Backed by the values dictionary that the server emits. |
| Notification | `abp.notify.{success,info,warn,error}` | Pluggable ŌĆö Toastr provider lives in `@abp/toastr`. |
| Message | `abp.message.{info,success,warn,error}` and a private `_showMessage` | Pluggable too ŌĆö SweetAlert2 provider replaces it. |
| UI | `abp.ui.block/unblock`, `abp.ui.setBusy/clearBusy` | Used by the loading spinner and form-submit guard. |
| Event bus | `abp.event.{on,off,trigger}` | A simple pub/sub keyed by event name (see `abp.event = (function () { ŌĆ” })()` around line 455). |
| Utils | `abp.utils.{createNamespace,replaceAll,formatString,...}` | C#-style `string.format` helper for use from Razor. |
| Security | `abp.security.antiForgery.{tokenCookieName,tokenHeaderName}` | Defaults `XSRF-TOKEN` / `RequestVerificationToken`. |
| Clock | `abp.clock.{kind,trySetBrowserTimeZoneToCookie,toLocaleStringOptions}` | Listens for `abp.configurationInitialized` to settle the time-zone cookie. |
| Features | `abp.features.{values}`, `abp.globalFeatures.{enabledFeatures}` | Feature toggle store. |

### Pub/sub example

The event bus is the most-used API outside of `abp.localization`:

```javascript
abp.event.on('abp.configurationInitialized', function () {
  // happens once the server-side AbpApplicationConfiguration is fetched
  if (abp.clock.trySetBrowserTimeZoneToCookie) { /* ... */ }
});

abp.event.trigger('myapp.products.created', { id: 42 });
```

ABP also uses it internally ŌĆö line 853 of `abp.js` listens for
`abp.configurationInitialized` to apply the browser time-zone cookie.

## @abp/utils

`npm/packs/utils/` is an `ng-packagr` library that exports a single class:

```typescript
// npm/packs/utils/projects/utils/src/public-api.ts
export * from './lib/linked-list';
```

```json
// npm/packs/utils/package.json
"name": "@abp/utils",
"main":  "dist/bundles/abp-utils.umd.js",
"dependencies": { "just-compare": "^2.3.0" }
```

`linked-list.ts` exposes `ListNode<T>` and `LinkedList<T>` with typical methods
(`addHead`, `addTail`, `attach`, etc.). It is consumed by the Angular packages
under `npm/ng-packs/` ŌĆö notably `@abp/ng.core` for route-history and replace
strategies ŌĆö and is *not* loaded into the browser via `install-libs`.
